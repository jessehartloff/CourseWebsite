== The API

This API serves as a way for the everyone in the class to used a shared database. The API handles all the database interactions and exposes the following functionality through HTTP requests.

GET / (http://fury.cse.buffalo.edu/musicRatings)

The API contains minimal documentation at its root url above.

GET /getAllSongs (http://fury.cse.buffalo.edu/musicRatings/getAllSongs)

Making a GET request to /getAllSongs will return every song and rating that has been added using the API. Since this is a GET request it can easily be accessed through a web browser by clicking the link. This will also show the format used to return the information which will need to be parsed. For this assignment you will make requests to /getAllSongs and process the returned data.

POST /addSong (http://fury.cse.buffalo.edu/musicRatings/addSong)

Use /addSong to add a new song to the API's database. Check the documentation to see an example of the JSON format used to add a song. Any song added must have a valid YouTube video ID (this is not the YouTube link) of a video for the song. To add songs you must use the API key that has been emailed to you.

You will add songs as part of lab 10 (goo.gl/I44oPz). This lab has been released and can be completed before its designated lab section.

POST /rateSong (http://fury.cse.buffalo.edu/musicRatings/rateSong)

Use /rateSong to add rating to a song added by someone other than yourself (you rate your own songs when you add them). Check the documentation to see an example of the JSON format used to rate a song. Songs are identified by their YouTube ID's so there is no need to resend the title and artist while rating a song. To add songs you must use the API key that has been emailed to you.

You will rate songs as part of lab 10 (goo.gl/I44oPz). This lab has been released and can be completed before its designated lab section.
